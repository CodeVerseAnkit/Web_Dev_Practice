CORE WEB VITALS

CLS
Cumulative Layout Shift (CLS) is a key Core Web Vitals metric that measures a webpage's visual stability by calculating unexpected layout movements during page load. A low CLS guarantees a disruption-free user experience, preventing users from accidentally clicking wrong buttons or losing their reading place.

The CLS Score

CLS is a unitless value representing how much visible content shifts and how far it moves. Google classifies scores based on real-world user data over a 75th percentile of page visits:
Good: ≤ 0.1
Needs Improvement: > 0.1 and ≤ 0.25
Poor: > 0.25

LCP
Largest Contentful Paint (LCP) is a Core Web Vital that measures perceived load speed by marking the render time of the largest image or text block visible in the viewport. A good LCP score is 2.5 seconds or less from the moment the user navigates to the page.
Achieving a fast LCP directly affects search engine rankings and significantly lowers bounce rates. When a page loads, the browser continuously evaluates elements as they appear on the screen to determine the largest one.

LCP Score Thresholds

Good: 2.5 seconds or less
Needs Improvement: Between 2.5 and 4.0 seconds
Poor: Over 4.0 seconds

FID
First Input Delay (FID) is a former Google Core Web Vital metric that measured web page responsiveness and interactivity. Specifically, it quantified the time from when a user first interacted with a web page (e.g., clicking a link or tapping a button) to the time when the browser was actually able to begin processing event handlers in response to that interaction.

How FID was Scored

Good: 100 ms or less
Needs Improvement: Between 100 ms and 300 ms
Poor: Over 300 ms
